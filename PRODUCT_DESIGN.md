# 🧬 Project Morpho：边缘进化 — 产品设计说明书

> **版本**: v2.5 | **更新**: 2026-03-05 | **负责人**: CPO + Engineering Team

---

## 一、项目愿景

**Project Morpho** 不是一款传统的生存类游戏。

我们卖的不是生存压力，而是**"生命在极端环境下绽放的逻辑美学"**。

玩家扮演造物主，在 Cloudflare 全球边缘节点上养育一只由 JSON 基因组驱动的数字生命。其核心交互闭环为：

> **自然语言意图** → **AI 逻辑裁判（LLM）评估合理性** → **LLM 解析为合法 JSON Patch** → **Wasm 物理引擎硬上限校验** → **写入基因组，渲染更新**

每一次变异必须经历**输入 → AI 裁定 → 物理验证**的完整闭环，每一次灭绝都值得被铭记为**数字化石**，每一张物种截图都应该值得被设为手机壁纸。

**成功标准**：玩家愿意把自己物种的截图设为手机壁纸。

---

## 二、核心设计原则

| 原则 | 说明 |
|---|---|
| **结构重于数值** | 不定义"血量"，而是定义"骨骼密度""储能效率"等物理属性 |
| **质感即数据** | 生物状态通过材质（透光率、粗糙度）展现，严禁 UI 飘字和血条 |
| **光影即反馈** | 环境剧变通过全局光照色温剧变来呈现，营造"神迹降临"的仪式感 |
| **悲剧即美学** | 灭绝不是失败，而是一首关于尝试与牺牲的演化之诗 |

## 三、视觉语言："活体博物馆 (The Living Museum)"

我们的视觉参考是**《纪念碑谷》的极简几何留白** + **《企鹅岛》的治愈系观察交互**，但我们不抄袭，我们创造自己的语言：

### 1. 等轴测孤岛 (Isolated Isles)
每个物种不是生存在杂乱背景中，而是悬浮在一个**极简黑色圆柱展台**上。留白锁死玩家注意力到生物的细微变化。底部配有低调的呼吸光环。

### 2. 材质叙事 (Material Storytelling)
- **高膨胀压** → `transmission` 增大 → 生物变为半透明凝胶
- **高复杂度** → `metalness` 增大 → 表面具有锐利金属感
- 所有数值变化通过渲染管线体现，**不依赖任何 HUD 或飘字**

### 3. 光影联控 (Light as Feedback)

光影系统与**异步全球高压系统（第十一章）强绑定**。服务端每日推送新纪元环境参数时，全服客户端自动同步切换氛围，营造真正的"神迹降临"仪式感。

| 纪元环境 | 全局氛围（全服同步切换） |
|---|---|
| 凛冬纪元（极寒 / 冰冻） | 深蓝底光 + 青冰主光，生物笼罩极光冷辉 |
| 熔核纪元（高热 / 辐射） | 暗红警戒背景 + 橙黄点阵光，如末世火焰席卷 |
| 毒沼纪元（强酸 / 毒雾） | 幽绿环境光 + 荧光绿核心光，腐蚀性光泽笼罩全场 |
| 太古纪元（初始默认） | 极光紫底光 + 深蓝主光，宁静深海般的原初氛围 |

### 4. 语义明信片 (Semantic Postcards)
一键生成 1200×800 海报：左侧 3D 孤岛渲染截图，右侧配有基因组 JSON 摘要和 AI 撰写的物种档案句子。这是产品的**核心传播单元**。

## 七、本地开发快速上手

### 1. 克隆与安装
```bash
git clone <repo-url>
cd 孢子进化/project
npm install
```

### 2. 构建 Wasm 物理引擎
```bash
cd wasm-physics
npm install
npm run asbuild:release
# 产物位于 build/release.wasm，部署时需上传至 R2 Bucket 命名为 physics.wasm
```

### 3. 配置环境
在 `wrangler.toml` 中替换你的实际资源 ID：
- `database_id`：Cloudflare D1 数据库 ID
- `bucket_name`：R2 存储桶名称
- `index_name`：Vectorize 索引名称

### 4. 本地预览
```bash
cd project
npx wrangler dev
# 访问 http://localhost:8787
```

## 八、基因组 JSON 规范（速查）

```json
{
  "metadata": { "genome_id": "morpho-xxx", "generation": 0 },
  "morphology": {
    "hox_segments": [{ "type": "thorax", "scaling": { "radius": 0.5 } }],
    "surface": { "permeability": 0.05 }
  },
  "metabolism": { "energy_storage_capacity": 100, "diet_type": "photosynthetic" },
  "genetics": { "mutation_rate": 0.02, "semantic_memory": ["Survived cold-snap at Gen2"] }
}
```

> ⚠️ **CPO 核心约束**：所有数值（如 `scaling`）的 Wasm 验证层**必须有硬上限**（0.1 ~ 5.0），防止 AI 输出无限巨兽。

## 九、设计禁忌（CPO 避坑清单）

1. ❌ **严禁血条、数字飘字、进度条**——状态通过生物动态、颜色和呼吸频率暗示
2. ❌ **严禁直接抄袭纪念碑谷图像**——汲取其美学哲学，创造自己的视觉语言
3. ❌ **严禁 AI 自由输出无约束的基因组**——所有突变必须经过 Wasm 物理引擎数值上限校验（防巨兽）
4. ❌ **严禁跨越碳基生物学边界的非法变异**——不得进化出加特林机枪、钢铁履带、激光炮等非生物结构。LLM 层必须部署**安全护栏 (Safety Guardrails)**，在逻辑裁判阶段直接驳回，绝不传递至 JSON 生成环节（防止 Prompt Injection 攻击）
5. ✅ **每个基因组版本必须有 `genome_version` 字段**，以备未来算法升级时的兼容转译

## 十、核心交互机制：结构化变异输入 (Structured Mutation Input)

> **原文档盲区补充**：玩家通过什么操作来改变基因组？

### 设计思路
玩家不需要手写 JSON，也不需要了解代码。我们引入**「部位 + 意图」的双轴输入系统**：

```
┌─────────────────────────────────────────┐
│  选择变异部位：[ 表皮 ▼ ]               │
│                                         │
│  描述意图（100字以内）：                 │
│  "在表皮聚集感光细胞，形成原始眼点，    │
│   对强光产生本能的趋向行为"             │
│                                         │
│           [ 🧬 提交变异申请 ]           │
└─────────────────────────────────────────┘
```

### 技术链路
1. **前端**：下拉菜单限制部位范围（`morphology` / `metabolism` / `genetics`），文本框限制 100 字
2. **Worker AI**：将「部位 + 意图」转化为合法的 JSON Patch，并约束在该部位的字段范围内
3. **逻辑裁判**（见第十三章）：先评估生物学合理性，通过后才送入 Wasm 验证
4. **Wasm 引擎**：执行硬性物理上限校验，最终写入基因组

### 关键设计价值
- **降低门槛**：把代码操作变成自然语言创作
- **防止 Prompt 注入**：下拉菜单锁定目标字段，杜绝 AI 被诱导修改无关字段
- **增加策略深度**：有限的输入次数迫使玩家深思熟虑每次「申请」

## 十一、生存验证机制：异步全球高压系统 (Asynchronous Global Events)

> **原文档盲区补充**：环境如何变化？何时变化？变化后物种如何应对？

### 全球时间轴
系统维护一条**全球时间轴**，每 **24 小时现实时间 = 1 个游戏纪元**：

```
纪元 0 ──→ 纪元 1 ──→ 纪元 2 ──→ 纪元 3 …
  │            │            │
  │       陨石寒冬         超级火山喷发
  │       含氧量 -30%      温度 +80°C
  │
  └── 玩家在此期间进行变异，赶在灾变前完成适应
```

### 判定逻辑
灾变时，系统将物种的物理参数与环境阈值比对，不达标则触发灭绝：

| 物种属性 | 环境要求 | 结果 |
|---|---|---|
| `thermal_regulation: 0.5` | 温度耐受 ≥ 0.8 | ❌ 灭绝 → 数字化石 |
| `surface.permeability: 0.02` | 强酸渗透率 ≤ 0.05 | ✅ 存活 |

### 技术实现
- **服务端 Cron Trigger**：Cloudflare Workers 定时任务每 24 小时推送新的环境参数至全局 `BIOME_STATE` Durable Object
- **批量判定**：Worker 遍历所有活跃物种 DO，调用 `LifeCycleManager.runExtinctionCheck()` 方法进行裁决
- **灭绝处理**：将灭绝物种序列化为 `fossil` 状态写入 D1，触发 AI 生成墓志铭并 WebSocket 推送给玩家

## 十二、商业化与变现模型 (Monetization Strategy)

> **原文档盲区补充**：如何在前中期建立清晰的变现路径？

### 核心思路：用 API 成本驱动内购动力

LLM 调用、高清 AI 绘图、图鉴生成都有真实的边际成本。内购设计需以此为锚点。

### 变现层级

| 层级 | 用户类型 | 权益 |
|---|---|---|
| **免费探索者** | 新用户 | 每日 3 次变异申请，低分辨率素描图鉴 |
| **月卡演化者** | 轻付费 | 每日 15 次申请，高清 AI 图鉴（含 GIF 动图），访问数字化石馆 |
| **季卡研究员** | 中付费 | 每日 50 次申请，优先队列，解锁竞争赛季排名 |
| **创世权柄** | 重付费大 R | 购买私服星球，自定义灾变时间表和环境极端参数 |

### 核心内购点

```
1. 加速血清 (Evolution Serum)
   └── 购买后获得额外当日演化次数（×5、×10 包）

2. 史诗图鉴 (Epic Codex)
   └── 单次购买，调用高阶 AI 模型生成精美静图/GIF 图鉴
   └── 自动生成 1200×800 语义明信片，附带物种史诗配文

3. 创世权柄 (Genesis Authority)
   └── 限量出售，拥有者可在私有星球上自定义灾变设定
   └── 其他玩家可选择将物种「迁徙」至该星球接受挑战

4. 化石复苏 (Fossil Revival)
   └── 付费将历史灭绝物种以「隐性基因」形式重新投入基因库
```

## 十三、逻辑裁定引擎 (AI Adjudication Engine)

> **原文档盲区补充**：如何判断变异的「生物学合理性」，防止荒谬输入？

### 两层验证模型

```
玩家输入：「让眼睛直接进化出激光发射器」
        │
        ▼
┌───────────────────────────────┐
│  层 1：逻辑裁判（大语言模型） │  → 合理性评分: 8/100 ❌
│  评估碳基生物进化规律          │  → 拒绝，返回建议："可尝试
│  · 是否符合自然界案例参照      │     改为「进化出生物电荷放电器官」"
│  · 是否存在材料转化的中间路径  │
└───────────────────────────────┘
        │（仅评分 ≥ 60 时通过）
        ▼
┌───────────────────────────────┐
│  层 2：Wasm 物理内核（硬规则）│  → 数值校验：半径、压力、复杂度
│  · 数值上限校验                │  → 通过后写入基因组
│  · ATP 代价计算                │
│  · 熵减约束                    │
└───────────────────────────────┘
```

### 裁判提示词设计原则

```
你是一位严谨的演化生物学家级裁判。
评估以下变异意图是否对碳基生命合理（评分 0-100）：

[意图]：{玩家输入}
[当前物种形态]：{genome_summary}

输出 JSON：
{
  "score": 75,
  "verdict": "PASSED",
  "reasoning": "...",
  "suggestion": "..."  // 仅当拒绝时提供替代建议
}
```

### 边界案例处理

| 输入 | 评分 | 理由 |
|---|---|---|
| "进化出更厚的甲壳来抵御强酸" | 92 ✅ | 自然界有大量案例参照 |
| "进化出超声波感知" | 68 ✅ | 鲸鱼/蝙蝠有参考，路径合理 |
| "从植物性变为铁基生命" | 12 ❌ | 无中间路径，违反碳基约束 |
| "自发产生抗引力结构" | 5 ❌ | 违反物理法则 |
