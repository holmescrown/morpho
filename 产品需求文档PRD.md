# Project Morpho：边缘进化 (Complete PRD v4.0)

## 1. 产品综述 (Product Overview)

**产品名称**： Project Morpho (代号：边缘进化)

**核心愿景**： 摆脱传统的"数值加点"，利用 AI 语义逻辑 驱动真实的 生物结构演化。

**产品定位**： 部署在边缘侧（Cloudflare Workers）的去中心化、非线性生命演化沙盒。

**核心卖点 (USP)**：
- **逻辑突变**： 摆脱数值加点，实现基于解剖学逻辑的"结构进化"。
- **语义生存**： 玩家通过自然语言改变环境（如"由于超新星爆发，重力变为 3 倍"），AI 实时推演物种变异。
- **全球迁徙**： 物种在 Cloudflare 全球边缘节点间真实"迁徙"，形成去中心化的数字生态。
- **经济生态**： 基于 Evo-Points 的数据工厂，玩家贡献观察日志优化 AI 对生物逻辑的理解。

## 2. 核心机制设计 (Core Mechanics)

### 2.1 异速生长基因组 (The Allometric Genome)

放弃传统的生命值/攻击力，使用 Hox 基因编码系统：

- **结构参数**： 定义身体区段 (Segments)、对称性 (Symmetry) 和肢体比例。生物形态由 L-System (分形几何) 骨架定义。
- **演化逻辑**： AI 修改的是发育比例（Scaling Factor），而非简单的血量。进化是"权衡 (Trade-off)"，例如：为了获得飞行能力，必须大幅降低骨骼密度，增加代谢频率。
- **位运算碱基编码**： 采用 Bit-level encoding 技术，将基因序列编码为二进制数据，提高存储效率和变异计算速度。
- **马尔可夫链变异模型**： 模拟碱基的 Transition（转换）与 Transversion（颠换），呈现生物学上的"不均匀性"，形成"演化热点"。

### 2.2 双层决策引擎 (The Dual-Layer Engine)

我们要做的不是一个随机生成器，而是一个受物理学与进化学约束的数字实验室：

- **语义层 (Logic Layer)**： Workers AI (Llama 3.3) 推理突变逻辑，负责语义环境解析与逻辑突变推理。
- **验证层 (Shield Layer)**： Wasm 模块执行物理硬约束，过滤掉"激光眼"等非逻辑幻觉。

### 2.3 生存公式与代谢算法

引入代谢成本与环境压力的动态博弈：

**代谢平衡**：
$$E_{input} \ge E_{basal} + \sum (m_i \cdot c_i) + E_{locomotion}$$

**熵增阻尼**：
$$E_{entropy} = \gamma \cdot \text{Complexity}^n \cdot e^{-\lambda \cdot \text{Adaptability}}$$

**总能耗约束**：
$$\text{Cost}_{total} = \sum (E_{rep} + E_{trans}) + \gamma \cdot \text{Complexity}^n \cdot e^{-\lambda \cdot \text{Adaptability}}$$

器官越复杂，生存成本越高。环境压力越大，适应性要求越高。

### 2.4 三层能耗约束与"性状涌现"

#### 2.4.1 复制与翻译成本 (E_rep, E_trans)

- **位运算编码**： 利用 Bit-level encoding，Wasm 计算基因序列的长度。
- **繁殖能耗**： 序列越长，生物在繁殖时的 ATP 扣除就越多，自然限制了生物无限制叠加基因块的贪婪倾向。

#### 2.4.2 熵增惩罚与适应性对冲

- **复杂度惩罚**： 复杂度越高的生物（Complexity^n），如果没有相应的环境适应性（Adaptability），其熵增成本将指数级上升。
- **生存策略**： 解释了为什么"布满褶皱的小型球体"能存活，而"长满散热器的巨兽"会崩溃。

#### 2.4.3 性状涌现机制

- **集体适应**： 当一群生物集体为了躲避暴雨而演化出"疏水表皮"时，会形成明显的谱系分支（Clade）。
- **演化热点**： 某些基因区段更易突变，形成"演化热点"，加速特定性状的涌现。

### 2.5 基因调控网络 (GRN) 协议 v1.0

为了实现"环境鲁棒性"，我们在 JSON 基因组中定义启动子（Promoter）与表达逻辑。

#### 2.5.1 GRN 协议 JSON 示例

```json
{
  "grn_id": "regulatory_network_v1",
  "regulatory_elements": [
    {
      "element_id": "ST_01",
      "type": "promoter",
      "input_signal": "AMBIENT_TEMP",
      "activation_threshold": 35.0,
      "logic": "GT",
      "target_gene_blocks": ["HEAT_FOLD_GENE", "SWEAT_GLAND_GENE"]
    },
    {
      "element_id": "ST_02",
      "type": "suppressor",
      "input_signal": "ENERGY_LEVEL",
      "inhibition_threshold": 20.0,
      "logic": "LT",
      "target_gene_blocks": ["LOCOMOTION_SCALING"]
    }
  ],
  "gene_blocks": {
    "HEAT_FOLD_GENE": {
      "atp_maintenance_cost": 2.5,
      "morphology_patch": { "surface.texture": "convoluted", "scaling.radius": 1.2 }
    },
    "LOCOMOTION_SCALING": {
      "atp_maintenance_cost": 5.0,
      "morphology_patch": { "hox_segments.locomotor.scaling": 0.5 }
    }
  }
}
```

#### 2.5.2 GRN 工作原理

- **调控元件**： 包括启动子 (Promoter) 和抑制子 (Suppressor)，根据环境信号触发或抑制特定基因区块的表达。
- **输入信号**： 环境参数（如温度、能量水平）作为调控输入。
- **逻辑判断**： 基于阈值和逻辑运算符（如 GT >, LT <）决定是否激活基因。
- **基因区块**： 包含 ATP 维护成本和形态学补丁，直接影响生物的结构和代谢。

#### 2.5.3 ATP 消耗逻辑

- **基础维护成本**： 每个活跃的基因区块都有固定的 ATP 维护成本。
- **表达成本**： 基因激活时产生一次性 ATP 消耗。
- **环境响应成本**： 频繁的环境响应会增加额外的 ATP 消耗。

### 2.6 生态竞争算法 (Competition Logic)

基于"代谢效率"的异步博弈系统：

当物种 A 侵入物种 B 的 Durable Object 环境时，系统通过以下维度进行对撞：

- **摄取重合度**： 两个物种是否争夺同一种资源？重合度越高，竞争越激烈。
- **能效比**： 谁能以更少的代谢代价移动并捕食？
- **红皇后效应**： AI 会定期扫描两个物种的向量，生成一段"捕食者与猎物"的演化对抗日志，驱动双方持续进化。

### 2.7 自然选择补偿机制

不直接抹除畸形突变，而是生成带 AI 叙事的"演化悲剧"，增强玩家与生物之间的情感纽带。例如：

- 记录物种因"过度追求速度导致心脏负荷过大"而灭绝的故事
- 生成包含进化教训的 AI 叙事，如"这个物种的翅膀过于庞大，无法支撑其体重，最终从天空坠落"

## 3. 技术执行路径：从位运算到向量观测

### 3.1 高性能内核 (Rust/Wasm)

- **位运算模拟**： Wasm 负责每秒数万次的位运算，模拟碱基的 Transition（转换）与 Transversion（颠换）。
- **马尔可夫链模型**： 突变呈现出生物学上的"不均匀性"，例如某些基因区段更易突变，形成"演化热点"。
- **边缘端性能**： 利用 Rust/Wasm 在边缘端处理高频位运算，确保 Cloudflare Workers 性能达标。

### 3.2 物种大爆发观测 (Vector Embeddings)

- **向量化表示**： 利用 Cloudflare Vectorize，将每个物种的 GRN 状态向量化。
- **实时聚类**： 玩家可以在前端看到实时聚类图，当一群生物集体演化出特定性状时，会看到明显的谱系分支（Clade）在向量空间中形成。
- **演化趋势分析**： 通过向量空间分析，预测物种的演化方向和潜在的适应策略。

## 4. 经济与数据生态系统 (Evo-Points System)

这不仅仅是一个游戏，更是一个人类偏好对齐 (RLHF) 的数据工厂：

### 4.1 突变能量 (Evo-Points)

作为系统内的"通用货币"，平衡 AI 调用成本：

- **获取方式**：
  - 生物在模拟中成功摄取资源或生存达到一定时间
  - 玩家撰写高质量的观察日志
  - 参与基因交易所交易
- **消耗方式**：
  - 触发 AI 突变推理
  - 进行跨服迁徙
  - 进行大规模环境干预
  - 购买独特的基因片段

### 4.2 观察日志 (Observation Log)

玩家撰写观察心得获取点数。这些日志本质上是高质量的人工标注数据，用于反馈并优化 AI 对生物逻辑的理解：

- **内容要求**： 描述生物的行为模式、环境适应策略、进化优势与劣势
- **审核机制**： AI 评估日志质量，高质量日志获得更多 Evo-Points
- **数据价值**： 用于微调 Workers AI 模型，提升突变逻辑的合理性

### 4.3 基因交易所

引入显性/隐性基因概念。玩家可以交易独特的 JSON 基因片段，形成一个类似"种畜市场"的 UGC 社区：

- **基因片段**： 玩家可以提取和交易特定的基因片段（如：翅膀结构、感官器官、防御机制）
- **基因评级**： 系统根据基因片段的适应度和独特性进行评级
- **防抄袭机制**： 利用 Vectorize 向量库检测基因片段的相似度，防止直接复制

## 6. 用户体验与交互 (User Journey)

### 6.1 实验室阶段 (Lab Phase)

- **播种**： 投入初始"数字孢子"。
- **引导突变**： AI 提供三个基于逻辑的进化选项（如：强化外壳、增大感知范围、优化消化系统）。
- **语义干扰**： 玩家输入："让这片海域变得极其粘稠。"
- **ATP 系统**： 生物在模拟中成功摄取资源或生存达到一定时间，会为玩家积累 ATP（能量单位），用于触发 AI 突变推理、跨服迁徙或大规模环境干预。

### 6.2 观测与社交阶段 (Social Phase)

- **物种档案**： AI 自动生成带有伪科学风格的《调查报告》，支持一键分享至社交媒体。
- **全球入侵**： 用户的物种被 Vectorize 推送到全球其他节点的玩家世界中，胜出者可获得"进化点"。
- **《物种起源》编年史**： 利用 Workers AI 将该物种的重要时刻（如：第一次长出眼睛、第一次在干旱中存活、第一次击败入侵者）串联成一段带图的 H5 页面。
- **数字化石奖励**： 灭绝的明星物种可以转化为"数字化石"背景，玩家分享时会带有："我的物种统治了东京节点 120 小时，最终因全球变暖灭绝"等强社交属性的文案。
- **全球进化树**： 前端展示一个由全球所有玩家贡献的实时进化树，第一个演化出关键特征的玩家 ID 会永久刻在该进化节点的里程碑上。
- **向量空间观测**： 玩家可以查看物种在向量空间中的聚类情况，观察演化趋势和谱系分支。

## 8. 商业模型与增长 (Monetization & Growth)

### 8.1 Freemium 模型

- **免费**： 维持 1 个物种，标准的进化速度和 Evo-Points 积累速率。
- **订阅 ($9.9/mo)**： 开启多模态（照片上传）进化、解锁多个"并行演化槽位"、独占 AI 叙事模型。

### 8.2 变现点

- **加速包**： 玩家可付费购买 Evo-Points 或"进化激素"来提升 AI 推理的复杂度和频率。
- **生态位扩展**： 免费用户仅限 1 个物种，付费用户可解锁多个"并行演化槽位"。
- **赛季通行证**： 每月开启一个"大灭绝事件"，推出限时挑战和专属奖励。
- **基因片段包**： 提供稀有基因片段的付费包，加速物种进化。

### 8.3 增长引擎

- **Viral Loop**： "我的客厅进化出了什么生物？"话题挑战，利用多模态功能驱动分享。
- **数字收藏**： 将具有里程碑意义的演化物种存入 R2，生成"数字化石"供他人参观。
- **全球排行榜**： 基于物种统治时间、击败入侵者数量等指标，设立全球排行榜，激发竞争心理。
- **数据贡献奖励**： 玩家贡献高质量观察日志获得额外 Evo-Points，形成正向循环。

## 9. 风险管控 (Risk Management)

### 9.1 性能瓶颈

- 限制单个 DO 实例的物种上限，将复杂模拟移至客户端（WebWorker）。
- 对免费用户的突变频率进行冷却时间 (CD) 限制，控制 AI 推理成本。
- 利用 Rust/Wasm 在边缘端处理高频位运算，确保 Cloudflare Workers 性能达标。

### 9.2 技术护栏补强

**三重过滤器**：

- **输入过滤**： 在 AI 推理前，利用 Workers AI 预置的安全模型拦截"原子弹"、"反重力"等破坏模拟逻辑的关键词。
- **Schema 约束**： 强制要求 LLM 输出符合特定的 JSON Schema（如：各肢体角度限制在 -90° 到 90° 之间）。
- **Wasm 物理审查**： 运行在 50ms 内的物理校验模块，判定生物是否符合热力学定律和基础生物学原理（如：重心是否稳定、代谢是否平衡）。

### 9.3 合规性

- 对玩家输入的语义环境进行敏感词过滤（利用 Workers AI 自带的安全护栏）。
- 确保生成的内容符合各地区的法律法规和文化习俗。
- 对基因交易所的内容进行审核，防止恶意或违法内容的传播。

## 11. 关键成功指标 (KPI)

- **用户留存**： 7 日留存率 ≥ 40%，30 日留存率 ≥ 20%。
- **付费转化**： 付费用户占比 ≥ 5%，平均客单价 (ARPU) ≥ $15/月。
- **社交传播**： 每个活跃用户平均分享次数 ≥ 2 次/月，分享后新用户转化率 ≥ 8%。
- **技术指标**： AI 推理平均响应时间 ≤ 2 秒，99% 请求响应时间 ≤ 5 秒。
- **生态健康**： 全球活跃物种数量 ≥ 10,000，物种灭绝率控制在 30% 以内（确保生态平衡）。
- **数据贡献**： 每日新增观察日志 ≥ 5000 条，高质量日志占比 ≥ 60%。
- **经济活跃**： 基因交易所月交易额 ≥ $50,000，Evo-Points 流通率 ≥ 80%。
