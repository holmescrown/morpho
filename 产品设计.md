# 🧬 Project Morpho：边缘进化 — 产品设计说明书

> **版本**: v2.5 | **更新**: 2026-03-05 | **负责人**: CPO + Engineering Team

---

## 一、项目愿景

**Project Morpho** 不是一款传统的生存类游戏。

我们卖的不是生存压力，而是**"生命在极端环境下绽放的逻辑美学"**。

玩家扮演造物主，在 Cloudflare 全球边缘节点上养育一只由 JSON 基因组驱动的数字生命。其核心交互闭环为：

> **自然语言意图** → **AI 逻辑裁判（LLM）评估合理性** → **LLM 解析为合法 JSON Patch** → **Wasm 物理引擎硬上限校验** → **写入基因组，渲染更新**

每一次变异必须经历**输入 → AI 裁定 → 物理验证**的完整闭环，每一次灭绝都值得被铭记为**数字化石**，每一张物种截图都应该值得被设为手机壁纸。

**成功标准**：玩家愿意把自己物种的截图设为手机壁纸。

## 二、核心设计原则

| 原则 | 说明 |
|---|---|
| **结构重于数值** | 不定义"血量"，而是定义"骨骼密度""储能效率"等物理属性 |
| **质感即数据** | 生物状态通过材质（透光率、粗糙度）展现，严禁 UI 飘字和血条 |
| **光影即反馈** | 环境剧变通过全局光照色温剧变来呈现，营造"神迹降临"的仪式感 |
| **悲剧即美学** | 灭绝不是失败，而是一首关于尝试与牺牲的演化之诗 |

## 三、视觉语言："活体博物馆 (The Living Museum)"

我们的视觉参考是**《纪念碑谷》的极简几何留白** + **《企鹅岛》的治愈系观察交互**，但我们不抄袭，我们创造自己的语言：

### 1. 等轴测孤岛 (Isolated Isles)
每个物种不是生存在杂乱背景中，而是悬浮在一个**极简黑色圆柱展台**上。留白锁死玩家注意力到生物的细微变化。底部配有低调的呼吸光环。

### 2. 材质叙事 (Material Storytelling)
- **高膨胀压** → `transmission` 增大 → 生物变为半透明凝胶
- **高复杂度** → `metalness` 增大 → 表面具有锐利金属感
- 所有数值变化通过渲染管线体现，**不依赖任何 HUD 或飘字**

### 3. 光影联控 (Light as Feedback)

光影系统与**异步全球高压系统（第十一章）强绑定**。服务端每日推送新纪元环境参数时，全服客户端自动同步切换氛围，营造真正的"神迹降临"仪式感。

| 纪元环境 | 全局氛围（全服同步切换） |
|---|---|
| 凛冬纪元（极寒 / 冰冻） | 深蓝底光 + 青冰主光，生物笼罩极光冷辉 |
| 熔核纪元（高热 / 辐射） | 暗红警戒背景 + 橙黄点阵光，如末世火焰席卷 |
| 毒沼纪元（强酸 / 毒雾） | 幽绿环境光 + 荧光绿核心光，腐蚀性光泽笼罩全场 |
| 太古纪元（初始默认） | 极光紫底光 + 深蓝主光，宁静深海般的原初氛围 |

### 4. 语义明信片 (Semantic Postcards)
一键生成 1200×800 海报：左侧 3D 孤岛渲染截图，右侧配有基因组 JSON 摘要和 AI 撰写的物种档案句子。这是产品的**核心传播单元**。

## 七、本地开发快速上手

### 1. 克隆与安装
```bash
git clone <repo-url>
cd 孢子进化/project
npm install
```

### 2. 构建 Wasm 物理引擎
```bash
cd wasm-physics
npm install
npm run asbuild:release
# 产物位于 build/release.wasm，部署时需上传至 R2 Bucket 命名为 physics.wasm
```

在 `wrangler.toml` 中替换你的实际资源 ID：
- `database_id`：Cloudflare D1 数据库 ID
- `bucket_name`：R2 存储桶名称
- `index_name`：Vectorize 索引名称

### 4. 本地预览
```bash
cd project
npx wrangler dev
# 访问 http://localhost:8787
```

## 八、基因组 JSON 规范（速查）

```json
{
  "metadata": { "genome_id": "morpho-xxx", "generation": 0 },
  "morphology": {
    "hox_segments": [{ "type": "thorax", "scaling": { "radius": 0.5 } }],
    "surface": { "permeability": 0.05 }
  },
  "metabolism": { "energy_storage_capacity": 100, "diet_type": "photosynthetic" },
  "genetics": { "mutation_rate": 0.02, "semantic_memory": ["Survived cold-snap at Gen2"] }
}
```

> ⚠️ **CPO 核心约束**：所有数值（如 `scaling`）的 Wasm 验证层**必须有硬上限**（0.1 ~ 5.0），防止 AI 输出无限巨兽。

## 九、设计禁忌与灵魂约束（CPO 避坑与编码宪法）

1. ❌ **严禁数值 UI**：严禁血条、数字飘字、进度条——状态通过生物动态、颜色和呼吸频率暗示。
2. ❌ **严禁非生物变异**：不得进化出加特林机枪、钢铁履带、激光炮等非生物结构。LLM 必须部署**安全护栏 (Safety Guardrails)**直接阻断。
3. ❌ **严禁跨越极限约束**：所有突变必须经过 Wasm 物理引擎的 Scaling 硬上限校验（防工业化巨兽）。
4. ❌ **反完美对称 (Soul)**：模型顶点(Vertex Offset)必须带有 5%-15% 的随机微小抖动，极度排斥工业平滑感。
5. ❌ **严禁纯色材质 (Soul)**：所有材质必须带有特定的 Noise 噪声或 BRDF 高光微结构处理，必须“去塑料感”。
6. ❌ **严禁 I/O 贪婪 (Soul)**：本地特征化大文件读取必须切断全盘扫描，限定采样模式并在 500ms 内释放流通道。

## 十、核心交互机制：结构化变异输入 (Structured Mutation Input)

> **原文档盲区补充**：玩家通过什么操作来改变基因组？

### 设计思路
玩家不需要手写 JSON，也不需要了解代码。我们引入**「部位 + 意图」的双轴输入系统**：

```
┌─────────────────────────────────────────┐
│  选择变异部位：[ 表皮 ▼ ]               │
│                                         │
│  描述意图（100字以内）：                 │
│  "在表皮聚集感光细胞，形成原始眼点，    │
│   对强光产生本能的趋向行为"             │
│                                         │
│           [ 🧬 提交变异申请 ]           │
└─────────────────────────────────────────┘
```

### 技术链路
1. **前端**：下拉菜单限制部位范围（`morphology` / `metabolism` / `genetics`），文本框限制 100 字
2. **Worker AI**：将「部位 + 意图」转化为合法的 JSON Patch，并约束在该部位的字段范围内
3. **逻辑裁判**（见第十三章）：先评估生物学合理性，通过后才送入 Wasm 验证
4. **Wasm 引擎**：执行硬性物理上限校验，最终写入基因组

### 关键设计价值
- **降低门槛**：把代码操作变成自然语言创作
- **防止 Prompt 注入**：下拉菜单锁定目标字段，杜绝 AI 被诱导修改无关字段
- **增加策略深度**：有限的输入次数迫使玩家深思熟虑每次「申请」

## 十一、生存验证机制：异步全球高压系统 (Asynchronous Global Events)

> **原文档盲区补充**：环境如何变化？何时变化？变化后物种如何应对？

### 全球时间轴
系统维护一条**全球时间轴**，每 **24 小时现实时间 = 1 个游戏纪元**：

```
纪元 0 ──→ 纪元 1 ──→ 纪元 2 ──→ 纪元 3 …
  │            │            │
  │       陨石寒冬         超级火山喷发
  │       含氧量 -30%      温度 +80°C
  │
  └── 玩家在此期间进行变异，赶在灾变前完成适应
```

### 判定逻辑
灾变时，系统将物种的物理参数与环境阈值比对，不达标则触发灭绝：

| 物种属性 | 环境要求 | 结果 |
|---|---|---|
| `thermal_regulation: 0.5` | 温度耐受 ≥ 0.8 | ❌ 灭绝 → 数字化石 |
| `surface.permeability: 0.02` | 强酸渗透率 ≤ 0.05 | ✅ 存活 |

### 技术实现
- **服务端 Cron Trigger**：Cloudflare Workers 定时任务每 24 小时推送新的环境参数至全局 `BIOME_STATE` Durable Object
- **批量判定**：Worker 遍历所有活跃物种 DO，调用 `LifeCycleManager.runExtinctionCheck()` 方法进行裁决
- **灭绝处理**：将灭绝物种序列化为 `fossil` 状态写入 D1，触发 AI 生成墓志铭并 WebSocket 推送给玩家

## 十三、逻辑裁定引擎 (AI Adjudication Engine)

> **原文档盲区补充**：如何判断变异的「生物学合理性」，防止荒谬输入？

### 两层验证模型

```
玩家输入：「让眼睛直接进化出激光发射器」
        │
        ▼
┌───────────────────────────────┐
│  层 1：逻辑裁判（大语言模型） │  → 合理性评分: 8/100 ❌
│  评估碳基生物进化规律          │  → 拒绝，返回建议："可尝试
│  · 是否符合自然界案例参照      │     改为「进化出生物电荷放电器官」"
│  · 是否存在材料转化的中间路径  │
└───────────────────────────────┘
        │（仅评分 ≥ 60 时通过）
        ▼
┌───────────────────────────────┐
│  层 2：Wasm 物理内核（硬规则）│  → 数值校验：半径、压力、复杂度
│  · 数值上限校验                │  → 通过后写入基因组
│  · ATP 代价计算                │
│  · 熵减约束                    │
└───────────────────────────────┘
```

### 裁判提示词设计原则

```
你是一位严谨的演化生物学家级裁判。
评估以下变异意图是否对碳基生命合理（评分 0-100）：

[意图]：{玩家输入}
[当前物种形态]：{genome_summary}

输出 JSON：
{
  "score": 75,
  "verdict": "PASSED",
  "reasoning": "...",
  "suggestion": "..."  // 仅当拒绝时提供替代建议
}
```

### 边界案例处理

| 输入 | 评分 | 理由 |
|---|---|---|
| "进化出更厚的甲壳来抵御强酸" | 92 ✅ | 自然界有大量案例参照 |
| "进化出超声波感知" | 68 ✅ | 鲸鱼/蝙蝠有参考，路径合理 |
| "从植物性变为铁基生命" | 12 ❌ | 无中间路径，违反碳基约束 |
| "自发产生抗引力结构" | 5 ❌ | 违反物理法则 |

## 十五、单人跑通阶段开发摘要 (Single Player Viable Loop)

本阶段由静态视觉演示升级为具备玩法闭环的“活体实验室”，奠定了 Project Morpho 的长远游玩深度：

### 1. “双轨制”运行逻辑 (Sandbox vs. Game Loop)
- **博物馆展示态 (Sandbox)**：允许玩家在前端 UI 将模型瞬间切换至各类高级“Hox 原型”（如水母、龙虾、章鱼），旨在让体验者快速感知游戏的终极视觉张力。**防抖机制：沙盒内的变异意图与物理反馈仅在前端缓存计算，不会上传至全球 Vectorize 节点，保持数据库纯洁性**。
- **正式游戏态 (Genesis)**：所有玩家的正式存档绑定为单一初始状态：`{"type": "embryo", "scaling": 0.2}`。模型表现为带脉冲光晕的微小卵泡，一切高级器官（如拨水鳍、口器）只能通过 AI 天然语言交流 + Wasm 硬裁决，按时间线一步步注入。

### 2. 多物种 Hox 原型库 (Archetype Engine)
摒弃了早期的单纯“单孢子渲染”，我们内置了底层 `buildArchetype` 方法进行程序化生成：
- **水母 (Medusa)**：高透光率 (`transmission: 0.9`)、低刚性，依靠中央点光源与伞罩物理收缩正弦律动前进。
- **节肢龙虾 (Arthropod)**：低透光率、高金属感 (`metalness: 0.8`)，具备分段装甲，游动依赖刚性双螯与划水肢的摆动。
- **软体章鱼 (Cephalopod)**：红光伪装变异体，由高度柔和且随体态拉伸变形的长触手拖曳驱动。

### 3. 环境感官彻底“去 UI 化” (Sensory Mapping)
- **视觉压迫轴 (Visual Depth Sync)**：废弃屏幕上的 Y 轴高程数字显示。玩家的 $Y$ 轴坐标通过全局场景总线联动重绘：
  - 浅海层 ($Y > -50$)：高可见度深蓝 `#003366`，底部带有流动的动态水波焦散。
  - 午夜深渊 ($Y < -200$)：焦散被彻底关闭，空间渐变为深渊黑 `#000510`，除生物的自体发光外，没有任何环境反射。
- **听觉滤波器 (Data-Driven Audio)**：利用 Web Audio API 创建了持续的海洋白噪音循环，并串联了一个**动态低通滤波器 (Low-pass Filter)**。下潜越深，高频被切断越多，声音越发闷涩沉重；在突破阈值上限时，还会随机传出尖锐的“结构挤压碎裂音”，构建出教科书级的深水恐惧代入感。
- **高压物理锁**：Wasm 引擎在深水区将强制接管并缩小生物的 `Scaling` 基础系数，使模型表现出受压紧缩的本能肢体反应。

### 4. 资产化接口 (Semantic Postcards Ready)
预留了前端 `takePhotograph` 等轴测 1200x800 定制快照接口。该方法被设计为截断所有背景 HTML 标签，只对 `ArchetypeBuilder` 渲染出的最高 PBR 精度的物体进行拍照（如龙虾的刚性反光、水母的透光肌理）。这将用于后端 Cloudflare Vision (LLaVA) 对该截图进行语义识别，进而配对专属的 AI 铭文，构成产品的核心裂变分享单元。

### 5. 循环闭环：捕食与死亡引擎 (Predation & Death Engine)
为了实现完整的生存回路，我们引入了生理反馈闭环：
- **萤光捕食系统 (Algae Predation)**：海洋中随机散布着「深海萤光食物团（Algae）」。生物与其碰撞后可补充 15% ATP 能量。捕食不仅是数值的变化，还会触发核心材质的「瞬时增亮」光爆反馈。
- **死亡引擎 (The Death Engine - 90/10 Roguelike)**：
    - **命运骰子 (RNG Roll)**: 当 ATP 归零时，触发 0-100 概率判定。
    - **90% 溶解 (Reset)**: 模型化作光点消散，重置为原始胚胎 (`{"type": "embryo"}`)。
    - **10% 奇迹 (Legacy)**: 触发“基因降级保护”，随机保留 1-2 个核心基因突变（如抗酸皮层），退化为带金缮裂纹的残缺老兵形态。
- **化石记录 (The Fossil Record)**：每次死亡自动抓拍快照，生成记录存活时间、代号与死因的数字化石明信片。

## 十六、活体博物馆：沉浸式五阶段全交互设计

为了确保用户从打开网页到在深海捕食的第一分钟不流失，我们将交互包装为一次“意识上传”过程，分为以下五个阶段：

### 第一阶段：意识同步 (Loading & Login)
建立“造物主”的代入感。
1. **视觉**：全黑屏幕中心一个紫色（#8800ff）的呼吸光环，其缩放频率与 Web Audio 的低频心跳音同步。
2. **毛玻璃登录**：浮现极简登录框，背景带有模糊的深海流体动力学效果。
3. **交互与音效**：点击“同步意识”后，紫色光环产生强力波纹扩散转场，低音（Sub-bass）转化为持续的深海背景白噪音。

### 第二阶段：物种起源 (Selection & Genesis)
建立长期目标，从胚胎启程。
1. **原型展示**：展台上依次滑动切换展示水母（Medusa，透光率 0.9）、龙虾（Arthropod，金属感 0.8）或章鱼（Cephalopod）。
2. **胚胎觉醒**：点击“开启演化”后，高级形态崩解为粒子，展台中心只剩下一个 0.2 系数的原始胚胎（Embryo）。

### 第三阶段：下潜与洗礼 (The Descent)
从静止观察到动态游动的无缝过渡。
1. **环境转换**：展台下沉，镜头平滑转为第三人称追踪视角。背景色从纯黑插值为深蓝（#003366），焦散波纹流过生物表面。
2. **感官变化**：启动动态低通滤波器（声音变闷），WASD 控制激活，生物表现出受水压和阻力驱动的浮动感。

### 第四阶段：原初捕食与反馈 (Predation Loop)
通过观察生物状态判断生存情况。
1. **狩猎引导**：远处闪烁绿色萤光食物团（Algae）。
2. **去 UI 反馈**：移除数字提示。碰撞时核心产生“增亮脉冲”，体积物理性增加（Scaling +0.02）。
3. **AI 对话**：屏幕上方浮现半透明文字：“能量激增。细胞意识处于活跃态，可以执行初步变异申请。”

### 第五阶段：自然语言变异 (Mutation Interface)

Project Morpho 的灵魂逻辑。
1. **控制台**：双击生物或按 M 键唤出毛玻璃变异控制台。
2. **结构化变异**：选择部位（表皮/代谢/遗传）并输入意图（如“长出更厚实的几丁质外壳”）。
3. **裁定与呈现**：提交后表面流过“代码流光”。逻辑裁判（LLM）评估后，成功则实时形变/材质改变，失败则由 AI 给出生物学修正建议。

## 十七、桌面挂机模式：透明培养皿 (Desktop Companion Mode)

为了增强“养成感”与用户粘性，Project Morpho 支持通过 Tauri 实现的桌面悬浮窗模式。

### 1. 窗口特性 (Desktop Foundation)
- **透明培养皿**: 窗口背景完全透明 (`transparent: true`)，无边框 (`decorations: false`)，且始终置顶 (`always_on_top: true`)。
- **幽灵模式 (Ghost Mode)**: 支持点击穿透 (`ignore_cursor_events`)。玩家可以在其上方正常办公，生物如同悬浮在桌面壁纸上。
- **窗口缩放**: 主界面支持一键缩放至 300x300 的桌面角落窗口。

### 2. 三态感官驱动 (Sensory Feedback)
遵循“去 HUD”原则，桌面模式下不显示任何 UI，所有状态通过生命感传达：
- **能量匮乏 (Hunger)**: 当 $ATP < 20\%$ 时，生物发光核心产生低频、暗淡的“心律不齐”脉冲，材质从晶莹剔透转为干瘪暗淡。
- **环境不适 (Stress)**: 当全球纪元灾变（如熔核、大酸化）触发时，生物表面叠加红色或幽绿色的噪波点（StressShader），并伴随高频挣扎震颤。
- **突变觉醒 (Evolution)**: 接收到变异成功信号时，全模型流过高亮度扫描光栅，伴随轻微相机抖动，实时展示形态重构。

## 二十三、 随机性格行为矩阵 (Trait Behavior Matrix)

既然性格是随机生成的，它必须通过一套算法直接干预模型在桌面上的行为。
- **随机初始化 (Seeding)**：胚胎期注入随机 Hash 决定性格三维坐标 (如: {Curiosity: 0.8, Aggression: 0.2, Social_Need: 0.5})。
- **高好奇心 (Curiosity)**：随机游动频率更高，且游动路径倾向于靠近鼠标指针或系统屏幕活跃区域。
- **高侵略性 (Aggression)**：触碰鼠标时触发“警戒”红色脉冲，并执行快速反弹逃逸。
- **高社交需求 (Social_Need)**：长期停留在角落时 ATP 消耗加快（抑郁），迫使用户交互投喂。

## 二十四、 极限防卫与单机持久化 (Session Persistence & Instance Guard)

必须处理“用户关机”或“程序多开崩溃”等极端边界状态。
- **消化与溢出容错**：投喂低熵值文件导致虚弱（阻尼增大、透明）；一次性投喂巨量 ATP 引发基因架构强制不稳定变异。
- **状态心跳包护航**：每 5 分钟在 LocalStorage 中固化当前 ATP、性格、形态 Hash。
- **多进程领地拦截**：检测到已有实例，新进程将被警告或抛出强烈的全屏视觉噪波 (Noise Shader)，确保深渊的排他性。
- **系统窗口边界碰撞**：极速拖拽 Tauri 窗口撞击屏幕时，生物模型产生向内的挤压弹性形变。

## 二十五、 寂静与注魂协议 (Dormant & Soul-Infusion)

生与死在离线与上线时形成交互循环，复苏并非重置，而是代价卓绝的仪式。
- **时间流逝补偿裁决**：再次启动对比离线时间计算 ATP 损耗。若能量殆尽，启动即彻底灭绝。
- **残骸寂静化 (The Relic)**：模型脱去所有海雪粒子特效，停滞律动，变质为沉重的石化材质 (roughness 1.0)。控制台只留下一句冰冷的：“它在黑暗中停止了搏动”。
- **长按注魂交互**：玩家必须长按残骸中心 3 秒蓄力，中心裂缝开始蔓生暗金色的光路输送残余生命力。
- **90/10 裁决法则**：90% 概率崩碎为尘埃彻底归零；10% 的概率固化为“带有永恒残缺金缮特征”的孤本活体模型复苏。
